# 微博舆情每日监控 - 定时任务使用指南

## 📋 任务概述

本定时任务每天自动从微博搜索页面抓取《王者荣耀世界》相关帖子(前5页约50-60条)，进行多维度舆情分析，并将结果同步到GitHub Pages报告的"用户需求追踪"板块。

## ⏰ 执行计划

- **执行时间**: 每天下午 15:05 (Asia/Shanghai)
- **下次执行**: 2026-04-10 15:05:00
- **任务ID**: `待创建`
- **启用状态**: ✅ 已启用

## 🎯 任务流程

### 步骤1：数据爬取（浏览器自动化）
- 打开微博搜索页面: https://s.weibo.com/weibo?q=王者荣耀世界&Refer=topic_weibo
- 注入Cookie实现免密登录（如需完整数据）
- 通过URL分页参数(`?page=N`)获取前5页数据
- 每页模拟滚动加载触发瀑布流内容
- 提取帖子正文、作者、发布时间等信息
- 保存原始数据为JSON格式: `weibo_posts_latest.json`

### 步骤2：数据分析
对收集的帖子进行智能分类统计：
- **分类体系**（9类标签）:
  - 代练代肝：代练、代肝、代打等服务类需求
  - 道具交易：出售、求购、装备、账号等交易行为
  - 组队社交：找队友、公会招募、CP寻找等社交需求
  - 游戏攻略：教程、技巧、玩法介绍等内容消费
  - 游戏资讯：版本更新、活动预告、新闻资讯
  - 游戏内容：画质、配置、性能优化等技术讨论
  - 同人创作：Cosplay、画作、视频、小说等UGC
  - 游戏技术：BUG反馈、系统功能咨询
  - 水贴/其他：打卡、日常分享、感叹疑问等

- **核心发现提取**:
  - 服务类需求占比（代练代肝 + 道具交易）
  - 社交属性强度（组队社交类占比）
  - 内容消费活跃度（攻略 + 资讯合计占比）

- **典型帖子示例**: 为每个主要类别选取2条代表性帖子

### 步骤3：更新报告
- 读取 `wangzhe_report/index_with_tabs.html`
- 在"用户需求追踪"日期切换按钮中添加今天日期
- 创建今天日期的内容区块，填充微博分析数据
- 保持历史日期内容不变，新日期设为active状态

### 步骤4：提交推送
```bash
cd wangzhe_report
git add index_with_tabs.html
git commit -m "添加MM月DD日微博舆情分析数据"
git push
```

### 步骤5：交付产物
- `weibo_posts_latest.json` - 原始帖子数据
- `weibo_analysis_result.json` - 分析结果（含分类分布、核心发现、典型示例）
- `index_with_tabs.html` - 已更新的GitHub Pages报告

## ⚠️ 注意事项

1. **Cookie有效期**: 
   - Cookie有效期约7天，需定期更新
   - 未登录状态下仅能获取部分帖子文本，作者、时间及互动数据缺失
   - 如数据不完整，需提供最新Cookie字符串

2. **浏览器自动化**: 
   - 微博有较强的反爬策略，避免短时间内频繁请求
   - 建议在两次抓取之间间隔至少5分钟
   - 如遇验证码或登录拦截，需等待自动通过或手动处理

3. **数据量限制**: 
   - 建议单次抓取不超过100条帖子（前5页）
   - 如需更多数据，可分多次执行并合并结果

4. **DOM结构变化**: 
   - 微博前端可能更新，导致选择器失效
   - 如提取失败需重新观察页面结构并调整选择器
   - 评论数等关键字段需直接观察DOM确认格式

5. **HTML结构**: 
   - 确保div标签配对正确，避免破坏页面布局
   - 只更新"用户需求追踪"板块，不影响"竞品动态追踪"等其他板块

6. **Git推送**: 
   - Git凭证已通过macOS钥匙串缓存，无需手动输入Token
   - 如果推送失败，记录错误信息并通知用户

7. **编码兼容性**: 
   - JSON文件使用UTF-8编码，确保中文正常显示
   - 如需导出CSV，必须添加UTF-8 BOM以确保Excel兼容

## 🆘 故障排查

### 任务未执行
1. 检查任务是否启用: `use_cron(action="list")`
2. 查看任务日志和错误信息
3. 确认工作区路径正确

### Cookie过期
1. 症状: 登录后仍显示未登录状态，或缺少作者、时间等字段
2. 解决: 用户提供最新Cookie字符串
3. 更新位置: 定时任务prompt中的Cookie字段

### 数据抓取失败
1. 检查网络连接
2. 确认微博URL可访问
3. 尝试手动执行测试: `use_cron(action="run", job_id="<任务ID>", wait=true)`
4. 检查是否触发反爬机制（验证码、登录拦截）

### HTML更新失败
1. 检查文件路径是否正确
2. 确认Git配置和权限
3. 验证HTML结构完整性（div标签配对）
4. 检查正则表达式是否匹配当前HTML结构

### Git推送失败
1. 检查Git远程仓库配置: `git remote -v`
2. 确认Token是否有效
3. 查看Git日志: `git log --oneline -5`
4. 检查是否有未提交的变更冲突

### 反爬拦截
1. 症状: 页面返回验证码、空白或登录提示
2. 解决: 
   - 增加等待时间，减少请求频率
   - 提供有效Cookie实现免密登录
   - 如持续失败，需用户手动处理验证码后继续

### 数据重复
1. 症状: 多页抓取时出现相同帖子
2. 解决: 在合并时进行去重处理（以正文前50字符+作者+时间为唯一标识）
