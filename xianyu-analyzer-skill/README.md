# 闲鱼数据分析 Skill 使用说明

## Skill 信息

- **名称**: xianyu-data-analyzer
- **ID**: a476db42-69b8-4a6b-b542-087b3ae18a2f
- **描述**: 闲鱼Excel数据分析与报告同步工具

## 快速开始

### 基本用法

```
用户: 分析这个闲鱼Excel文件并更新到报告
附件: 闲鱼-搜索关键词列表采集4.3.xlsx
```

Skill会自动:
1. 解析Excel文件获取商品数据
2. 分析商品类型分布、价格区间、ID命名特征
3. 生成符合规范的总结文本
4. 更新 `wangzhe_report/index_with_tabs.html` 中的"竞品三：闲鱼"区块
5. 提交并推送到GitHub Pages

### 指定日期

```
用户: 把4月2日的闲鱼数据更新到报告,Excel在 /tmp/xianyu_0402.xlsx
```

### 仅分析不更新

```
用户: 帮我分析一下这个闲鱼Excel的数据分布,先不更新报告
```

## 核心功能

### 1. 商品类型分析

自动识别以下类型:
- **ID/昵称交易**: 单字ID、双字ID、三字ID、四字及以上ID
- **账号交易**: 成品号、带皮肤账号等
- **周边/道具**: 海报、吧唧、装扮等

### 2. 价格区间统计

将商品价格划分为:
- ≤¥100
- ¥101-¥999
- ¥1,000-¥4,999
- ¥5,000-¥9,999
- ≥¥10,000

计算各区间占比,特别关注5000元以上高价商品比例。

### 3. ID命名特征

统计不同长度ID的分布:
- 单字ID数量及占比
- 双字ID数量及占比
- 三字ID数量及占比
- 四字及以上ID数量及占比

同时分析命名风格(霸气/中二、诗意/文学、可爱/萌系等)。

### 4. 报告同步

自动定位HTML报告中对应日期的"竞品三：闲鱼"区块,替换内容为生成的总结。

**输出格式示例**:
```html
<div class="competitor-card" style="margin-bottom: 20px;">
    <div class="competitor-name">竞品三：闲鱼</div>
    <h3 class="subsubsection-title">闲鱼市场分析报告</h3>
    <ul>
        <li><strong>ID经济占绝对主导：</strong>超过九成的商品为极品ID/昵称交易...</li>
        <li><strong>价格两极分化：</strong>普通ID多在¥5-¥50之间,而单字ID...</li>
        <li><strong>商品类型高度集中：</strong>ID/昵称交易占比90%以上...</li>
        <li><strong>命名特征多样化：</strong>单字ID(12.1%)、双字ID(38.4%)...</li>
    </ul>
</div>
```

## 技术实现

### 依赖工具

- `parse_file`: Excel文件解析
- `grep_search`: HTML文件内容定位
- `read_file`: 读取HTML文件上下文
- `modify_file`: 更新HTML文件内容
- `execute_shell`: Git操作

### 工作流程

```
1. parse_file 解析Excel
   ↓
2. 数据分析(类型/价格/命名)
   ↓
3. 生成总结文本
   ↓
4. grep_search 定位HTML目标位置
   ↓
5. modify_file 更新内容
   ↓
6. git add/commit/push 推送更新
```

## 注意事项

1. **数据隐私**: Excel文件可能包含敏感交易数据,分析完成后建议删除临时文件

2. **报告一致性**: 确保同一日期下的三个竞品(螃蟹/盼之/闲鱼)数据完整性,避免单独更新导致其他竞品数据丢失

3. **空状态处理**: 若某天无闲鱼数据,保留区块但显示"暂无新动态",不复制前一天数据

4. **总结规范**: 遵循"闲鱼部分仅列举主要商品类型,不呈现具体数字"的规范,除非用户明确要求详细数据

5. **Git原子性**: 每次更新应包含完整的日期区块,避免部分提交导致报告结构损坏

## 错误处理

### Excel解析失败
- 检查文件路径是否正确
- 确认文件格式为 `.xlsx` 或 `.xls`
- 尝试使用 `read_file` 查看文件是否损坏

### HTML定位失败
- 确认目标日期区块存在(如 `competitor-04-03`)
- 检查"竞品三：闲鱼"文本是否存在
- 若不存在,提示用户先创建该区块

### Git推送失败
- 检查网络连接
- 确认Git配置正确(user.name, user.email)
- 检查是否有未提交的更改冲突

### 数据不足或异常
- 若Excel中商品数量<10,提示数据量不足
- 若价格字段缺失,尝试从标题中提取
- 如实报告实际分析的商品数量
