## 📊 技术架构

### 数据流

```
微博搜索页面
    ↓ (浏览器自动化爬取)
weibo_posts_latest.json (原始数据)
    ↓ (智能分类分析)
weibo_analysis_result.json (分析结果)
    ↓ (HTML内容生成)
index_with_tabs.html (报告更新)
    ↓ (Git推送)
GitHub Pages (线上报告)
```

### 智能分类体系

9类标签体系：
1. **代练代肝**: 代练、代肝、代打等服务类需求
2. **道具交易**: 出售、求购、装备、账号等交易行为
3. **组队社交**: 找队友、公会招募、CP寻找等社交需求
4. **游戏攻略**: 教程、技巧、玩法介绍等内容消费
5. **游戏资讯**: 版本更新、活动预告、新闻资讯
6. **游戏内容**: 画质、配置、性能优化等技术讨论
7. **同人创作**: Cosplay、画作、视频、小说等UGC
8. **游戏技术**: BUG反馈、系统功能咨询
9. **水贴/其他**: 打卡、日常分享、感叹疑问等

### 核心发现提取

- **服务类需求占比**: 代练代肝 + 道具交易合计百分比
- **社交属性强度**: 组队社交类占比
- **内容消费活跃度**: 攻略 + 资讯合计占比

## ⚠️ 注意事项

1. **Cookie有效期**: Cookie有效期约7天，需定期更新；如数据不完整，需提供最新Cookie字符串
2. **反爬机制**: 微博有较强的反爬策略，避免短时间内频繁请求；建议在两次抓取之间间隔至少5分钟
3. **数据量限制**: 建议单次抓取不超过100条帖子（前5页）；如需更多数据，可分多次执行并合并结果
4. **DOM结构变化**: 微博前端可能更新，导致选择器失效；如提取失败需重新观察页面结构并调整选择器
5. **Git推送**: Git凭证已通过macOS钥匙串缓存，无需手动输入Token；如果推送失败，记录错误信息并通知用户
