## 🔄 日常更新流程

之后每天只需执行以下命令即可更新报告：

```bash
cd wangzhe_report

# 1. 编辑 data/ 下的 JSON 文件，添加新日期数据
# 例如：data/official_events.json, data/competitor_data.json, data/user_feedback.json

# 2. 生成新的 HTML 报告
python3 generate_report.py

# 3. 提交并推送
git add .
git commit -m "更新2026年3月31日数据"
git push
```

或者使用一键部署脚本：
```bash
./deploy.sh
```

推送后，GitHub Pages 会在 **1-2 分钟** 内自动刷新内容。

## ⚠️ 常见问题

### Q: 页面显示 404 怎么办？
A: GitHub Pages 首次部署需要 5-10分钟。请等待后刷新页面。如果超过 15 分钟仍显示 404，请检查：
1. 仓库是否为 Public（公开）
2. GitHub Pages 设置是否正确（Branch: main, Folder: /）

### Q: 如何修改报告样式？
A:编辑 `generate_report.py` 中的 CSS 样式部分，然后重新生成并推送。

### Q: 如何查看历史版本？
A: 访问 https://github.com/fungjiewen-collab/wangzhe_report/commits/main 查看所有提交记录。

### Q: Token 安全吗？
A: Token 已嵌入到 git remote URL 中。建议定期更换 Token，或使用 SSH key 替代 HTTPS 认证。
