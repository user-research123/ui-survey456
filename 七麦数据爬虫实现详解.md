## 项目概述

本文档详细说明如何从零开始实现一个七麦数据（qimai.cn）iOS游戏畅销榜爬虫，包括技术选型、实现步骤、遇到的挑战及解决方案。

**目标**: 抓取中国区iOS游戏畅销榜前20名，包含排名、游戏名称、开发商信息。

## 一、技术架构设计

### 1.1 核心挑战

七麦数据作为专业的移动应用数据分析平台，采用了多层反爬机制：

1. **登录验证**: 榜单数据需要登录后才能查看完整信息
2. **动态参数签名**: API请求需要`analysis`参数，该参数通过复杂算法动态生成
3. **会话管理**: 依赖多个Cookie字段维持登录状态（AUTHKEY、USERINFO、PHPSESSID等）
4. **前端渲染**: 数据通过JavaScript动态加载，非静态HTML

### 1.2 技术方案对比

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **直接API调用** | 速度快、资源占用少 | 需要破解签名算法 | ⭐⭐⭐ 推荐（需正确参数） |
| **浏览器自动化** | 模拟真实用户、绕过部分反爬 | 速度慢、易被检测 | ⭐⭐ 备选方案 |
| **中间人代理** | 可捕获真实请求 | 配置复杂 | ⭐ 高级用户 |

**最终选择**: 结合Cookie认证的API调用方式

### 步骤2: 获取有效Cookie

Cookie是访问七麦数据的关键，需要从已登录的浏览器会话中获取：

1. 在浏览器中访问 https://www.qimai.cn/ 并登录
2. 按 `F12` 打开开发者工具
3. 切换到 **Network** 标签
4. 刷新页面，找到任意API请求（如 `api.qimai.cn` 开头的请求）
5. 右键点击请求 → **Copy** → **Copy as cURL**
6. 从cURL命令中提取 `Cookie:` 后面的完整字符串

**关键Cookie字段**:
```
qm_check        - 请求签名验证
AUTHKEY         - 用户认证令牌
USERINFO        - 用户信息加密串
PHPSESSID       - PHP会话ID
synct/syncd     - 时间同步参数
```

### 步骤3: 分析API端点

通过浏览器开发者工具的Network面板，观察页面加载时的API调用：

**观察到的API端点**:

#### 4.2 Analysis参数生成（核心难点）

七麦数据的 `analysis` 参数生成算法未公开，但通过逆向工程可以推测其逻辑：

```python
import base64
import hashlib
import json

def generate_analysis(params_dict):
    """
    生成analysis参数（简化版，实际算法更复杂）
    
    推测的算法步骤:
    1. 将参数字典按键排序
    2. 拼接成查询字符串
    3. 添加时间戳和盐值
    4. 进行MD5或SHA加密
    5. Base64编码
    """
    # 步骤1: 排序并拼接参数
    sorted_params = sorted(params_dict.items())
    query_string = '&'.join([f"{k}={v}" for k, v in sorted_params])
    
    # 步骤2: 添加固定盐值（需要从JS代码中提取）
    salt = "xyz517cda96efgh"  # 示例盐值，实际需要逆向获取
    
    # 步骤3: 加密
    raw_data = f"{query_string}@{salt}"
    md5_hash = hashlib.md5(raw_data.encode()).hexdigest()
    
    # 步骤4: Base64编码
    analysis = base64.b64encode(md5_hash.encode()).decode()
    
    return analysis
```

**注意**: 上述代码是简化示例，真实的七麦加密算法涉及更多步骤和密钥，需要从网站的JavaScript源码中逆向提取。

    """解析榜单数据"""
    if response_json.get('code') != 10000:
        raise Exception(f"API错误: {response_json.get('msg')}")
    
    games = []

## 四、遇到的问题与解决方案

### 问题1: Cookie注入后仍然显示未登录

**现象**: 通过浏览器自动化注入Cookie后，页面仍显示"登录/注册后可查看更多数据"

**原因分析**:
1. `document.cookie` 设置的Cookie可能无法覆盖HttpOnly标记的Cookie
2. 某些Cookie字段（如AUTHKEY）可能是HttpOnly，只能通过HTTP响应头设置
3. Cookie可能有IP绑定或时效性限制

**解决方案**:
- 直接在Python脚本中使用requests库，通过headers传递Cookie
- 确保Cookie字符串完整，包含所有必要字段
- 检查Cookie是否过期（特别是synct时间戳）

### 问题2: API返回空数据（rankInfo为空数组）

**现象**: `{"code": 10000, "msg": "成功", "rankInfo": []}`

**原因分析**:
1. `analysis` 参数不正确或缺失
2. 请求参数组合不匹配（如date格式错误）
3. Cookie权限不足（可能需要VIP会员）

**解决方案**:
- 从浏览器Network面板复制真实的 `analysis` 参数值
- 确保所有必需参数都存在且格式正确
- 检查账户是否有查看榜单的权限

### 问题3: Analysis参数动态生成算法未知

**现象**: 每次请求的 `analysis` 值都不同，硬编码的值只能使用一次

**原因分析**:
- 七麦使用JavaScript混淆代码生成签名
- 签名算法包含时间戳、随机数等动态因素

**解决方案**:
- **短期**: 从浏览器捕获最新的 `analysis` 值直接使用
- **长期**: 逆向JavaScript源码，复现签名算法
  - 使用Chrome DevTools的Sources面板
  - 搜索关键词 "analysis"、"encrypt"、"sign"
  - 断点调试追踪参数生成过程

### 问题4: Browser Use的Evaluate功能失败

**现象**: 在七麦页面执行 `browser_use(action="evaluate")` 时持续报错

**原因分析**:
- 七麦网站可能有Content Security Policy (CSP) 限制
- 页面使用了复杂的框架结构，导致JavaScript注入失败

**解决方案**:
- 改用纯API调用方式，避免浏览器自动化
- 如果必须使用浏览器，考虑使用Selenium或Playwright等更成熟的工具
