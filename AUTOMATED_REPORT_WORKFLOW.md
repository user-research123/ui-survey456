## 关键脚本说明

### 1. `generate_daily_summary.py`
**功能**：日报总结生成器
- 从5个数据源加载当天数据
- 生成符合规范的6句话以内总结
- 自动更新HTML报告并推送到GitHub

**输出格式示例**：
```
官方活动方面：《王者荣耀世界》开启峡谷外相遇组队活动，《云·王者荣耀世界》抢先体验版正式上线并开放昵称抢注。
竞品方面：螃蟹账号QQ平台占比78%，ID交易以双字(47%)和单字(25%)为主，中位数价格999元；盼之商品类型多样化涵盖成品号、昵称及代肝，安卓QQ占比74%，双字ID占38%，中位数888元；闲鱼市场ID经济占绝对主导，超九成商品为极品ID/昵称交易，价格两极分化明显。
用户方面：在微博上，服务类需求（代练代肝+道具交易）合计占20%，组队社交占13%，成为核心关注点。
```

### 2. 数据持久化规范
所有爬虫任务必须将结果写入对应的JSON文件：
- `data/official_events.json` - 官方活动数据
- `data/pxb7_analysis.json` - 螃蟹账号分析数据
- `data/pzds_analysis.json` - 盼之平台分析数据
- `data/xianyu_analysis.json` - 闲鱼市场分析数据

## 总结生成规范

### 句子数量限制
- **总计不超过6句话**
- 如果某板块无新内容则省略该板块（如官方活动无更新则不写）

### 竞品独立总结
每个竞品必须有自己单独的总结段落：
- **螃蟹**：平台占比、ID类型分布、中位数价格
- **盼之**：商品类型、平台分布、命名特征、中位数价格
- **闲鱼**：ID经济主导地位、价格两极分化特征

### 数字格式规范
- 所有百分比显示为**整数**（如52%而非52.3%）
- 价格保留整数（如999元而非999.5元）

### 板块顺序
1. 官方活动方面（如有）
2. 竞品方面（螃蟹；盼之；闲鱼）
3. 用户方面（微博舆情核心关注点）

## 异常处理策略

### 数据源缺失
- 如果某个数据源缺失，该板块总结可以省略或显示"暂无新动态"
- 如果所有数据源都缺失，发送错误通知到钉钉群

### Git推送失败
- 记录错误日志但不中断流程
- 下次执行时会自动重试

### 盼之平台验证码拦截
- 启用Fallback策略：复用近期已成功获取的基准数据集
- 在报告中明确标注数据来源为"基准数据集（Fallback）"
