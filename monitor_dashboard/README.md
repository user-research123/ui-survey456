# 《王者荣耀：世界》市场AI监控汇总系统

## 📋 项目简介

这是一个自动化的市场监控报告生成系统，每天自动抓取游戏官方活动、竞品动态、用户需求等数据，生成美观的HTML报告。

## 🎯 功能特性

- ✅ **自动数据抓取**：集成七麦、微博、小红书、闲鱼等多平台数据源
- ✅ **美观可视化**：响应式设计，卡片式布局，渐变配色
- ✅ **定时更新**：每天上午9点自动生成最新报告
- ✅ **钉钉推送**：可选配置，生成后自动推送到团队群
- ✅ **历史存档**：每次生成的报告自动保存，便于对比分析

## 📁 项目结构

```
monitor_dashboard/
├── template.html          # HTML模板文件
├── generate_report.py     # 主脚本：生成HTML报告
├── fetch_data.py          # 数据抓取模块
├── output/                # 生成的报告输出目录
│   └── monitor_report_YYYYMMDD.html
└── README.md              # 使用说明
```

## 📊 报告内容

### 1. 核心总结
基于当天数据的关键洞察，包括市场趋势、用户痛点、竞品动态等。

### 2. 游戏官方事件/活动
时间线形式展示官方发布的重要信息和活动节点。

### 3. 竞品动态追踪
表格形式对比各平台（螃蟹、盼之、闲鱼）的商品类型、价格区间和热度标签。

### 4.用户需求追踪
分渠道（微博、小红书、闲鱼）展示用户关注点和需求分布。

## 🛠️ 技术栈

- **前端**: HTML5 + CSS3（响应式设计）
- **后端**: Python 3.9+
- **定时任务**: use_cron (cron表达式)
- **数据源**: 七麦数据、微博、小红书、闲鱼、螃蟹账号等

## 📝 注意事项

1. **数据源权限**：部分平台需要Cookie或登录态，请确保相关技能已正确配置
2. **网络访问**：确保服务器可以访问外部数据源
3. **存储空间**：定期清理旧的HTML文件，避免占用过多空间
4. **错误处理**：如果数据抓取失败，系统会自动降级使用示例数据

## 🔍 故障排查

### 报告未生成
- 检查定时任务状态：`use_cron(action=list)`
- 手动运行脚本测试：`python3 generate_report.py`
- 查看错误日志

###数据为空
-检查数据源连接
- 验证Cookie/登录态是否有效
- 查看 `fetch_data.py` 中的TODO注释，确认技能集成情况

### 样式异常
- 清除浏览器缓存
- 检查HTML文件编码（UTF-8）
-验证CSS语法
