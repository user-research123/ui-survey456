# 闲鱼平台每日商品分析 - 定时任务使用指南

## 📋 任务概述

本定时任务每天自动从闲鱼(goofish.com)抓取《王者荣耀世界》前100个商品信息，进行多维度数据分析，并将结果同步到GitHub Pages报告。

## ⏰ 执行计划

- **执行时间**: 每天下午 15:00 (Asia/Shanghai)
- **下次执行**: 2026-04-10 15:00:00
- **任务ID**: `b08e39ed-644b-4ae1-bfe0-64be145ce6d8`
- **启用状态**: ✅ 已启用

## 🎯 任务流程

### 步骤1：数据抓取（浏览器自动化）
- 打开闲鱼搜索页面: https://www.goofish.com/search?q=王者荣耀世界
- 注入Cookie实现免密登录
- 通过分页按钮点击获取100个商品（每页约30个，共4页）
- 提取商品标题、价格、链接等信息
- 保存原始数据为JSON格式

### 步骤2：数据分析
对100个商品进行统计分析：
- **价格分布**: 范围、中位数、高价商品占比(≥¥10,000)
- **价格区间**: 0-500、500-1000、1000-5000、5000-10000、10000以上
- **平台分布**: QQ、微信、未明确
- **命名特征**: 单字ID、双字ID、三字及以上
- **风格分类**: 诗意/文学类、霸气/中二类、可爱/萌系、明星/名人、其他

### 步骤3：更新报告
- 读取 `wangzhe_report/index_with_tabs.html`
- 在日期切换按钮中添加今天日期
- 添加今天日期的闲鱼分析内容区块
- 保持螃蟹、盼之等其他竞品内容不变

### 步骤4：提交推送
```bash
cd wangzhe_report
git add index_with_tabs.html
git commit -m "更新MM月DD日闲鱼平台商品分析数据"
git push
```

### 步骤5：交付产物
- `xianyu_data_YYYYMMDD.json` - 原始商品数据
- `xianyu_analysis_YYYYMMDD.txt` - 闲鱼平台商品数据分析报告
- `index_with_tabs.html` - 已更新的GitHub Pages报告

## ⚠️ 注意事项

1. **Cookie有效期**: 
   - Cookie有效期约7天，需定期更新
   - 当前Cookie最后更新时间：2026-04-09
   - 如登录失败，需提供最新Cookie字符串

2. **浏览器自动化**: 
   - 如果遇到验证码或超时问题，系统会自动增加等待时间
   - 分页按钮的ref ID每次可能不同，需动态获取

3. **数据完整性**: 
   - 如果实际收集的商品不足100个，会如实报告实际数量
   - 不要复制前一天的数据填充空白

4. **HTML结构**: 
   - 确保div标签配对正确，避免破坏页面布局
   - 只更新"竞品：闲鱼"区块，不影响其他竞品

5. **Git推送**: 
   - Git凭证已通过macOS钥匙串缓存，无需手动输入Token
   - 如果推送失败，记录错误信息并通知用户

6. **反爬处理**: 
   - 闲鱼可能有反爬限制，如遇验证码需等待自动通过或手动刷新
   - 增加等待时间可减少被拦截概率

## 🆘 故障排查

### 任务未执行
1. 检查任务是否启用: `use_cron(action="list")`
2. 查看任务日志和错误信息
3. 确认工作区路径正确

### Cookie过期
1. 症状: 登录后仍显示未登录状态
2. 解决: 用户提供最新Cookie字符串
3. 更新位置: 定时任务prompt中的Cookie字段

### 数据抓取失败
1. 检查网络连接
2. 确认闲鱼URL可访问
3. 尝试手动执行测试: `use_cron(action="run", job_id="b08e39ed-644b-4ae1-bfe0-64be145ce6d8", wait=true)`

### HTML更新失败
1. 检查文件路径是否正确
2. 确认Git配置和权限
3. 验证HTML结构完整性

### Git推送失败
1. 检查Git远程仓库配置: `git remote -v`
2. 确认Token是否有效
3. 查看Git日志: `git log --oneline -5`

### 反爬拦截
1. 症状: 页面返回验证码或空白
2. 解决: 增加等待时间，减少请求频率
3. 如持续失败，需用户提供新的Cookie或手动处理验证码
