#洛克王国：世界 - 小红书交易诉求监测系统使用指南

## 系统架构

本系统采用**"双轨监测 + AI 语义过滤"**策略，解决新品类漏检问题：

1. **精准轨**：使用高置信度关键词（如"出号"、"代肝"）捕获已知交易类型。
2. **发现轨**：使用宽泛意图词（如"蹲"、"求"、"有人要吗"）捕获新出现的交易黑话。
3. **AI 过滤器**：对发现轨数据进行语义分类，自动提取新的交易对象词。

## 目录结构

```
workspace/
├── xhs_monitor.py          # 核心处理脚本
├── README_XHS_MONITOR.md   # 本文档
├── xhs_raw_data/           # [需手动创建] 存放原始抓取数据的 JSON文件
└── xhs_processed_data/     # [自动创建] 存放处理后数据和周报
```

## 数据获取流程 (目前需半自动)

由于小红书反爬严格，目前建议采用以下两种方式之一获取原始数据：

### 方案 A：浏览器插件导出 (推荐)

1. 安装浏览器插件如 "Web Scraper"或 "Data Miner"。
2. 在小红书搜索页分别输入**精准关键词**和**发现关键词**。
3. 配置插件抓取字段：`标题`、`正文`、`作者`、`发布时间`、`链接`。
4. 将抓取结果导出为 JSON 格式，放入 `xhs_raw_data/` 目录。

### 方案 B：手动复制粘贴 (小规模测试)

1. 在小红书搜索相关关键词。
2. 手动复制帖子标题和正文。
3. 按照以下 JSON 格式保存为文件（如 `manual_20260327.json`）：

```json
[
  {
    "title": "帖子标题",
    "content": "帖子正文内容",
    "author": "作者昵称",
    "publish_time": "2026-03-27T10:00:00",
    "url": "https://www.xiaohongshu.com/..."
  }
]
```

## 运行脚本

确保已安装 Python 3.9+ 和 `requests` 库：

```bash
pip3 install requests
python3 xhs_monitor.py
```

## 定时任务配置

系统已配置每日上午 10:00 自动运行数据处理流程。你可以通过钉钉机器人接收每日新词简报。

## 新词迭代流程

1.脚本运行后，会在 `xhs_processed_data/` 生成 `new_words_YYYYMMDD_HHMMSS.txt`。
2. 人工审核新词，确认其为有效交易品类。
3. 将新词加入 `xhs_monitor.py` 中的 `precision_keywords` 列表。
4. 重新运行脚本或等待下次定时任务。

## 注意事项

- **噪音过滤**：脚本内置了排除词列表（如"攻略"、"教程"），可根据实际情况调整。
- **AI 分类**：目前使用规则模拟，后续可接入真实 LLM API 提高准确率。
- **钉钉推送**：确保网络环境可以访问钉钉 Webhook 地址。
