# 交易猫商品详情页UI改版用户研究方案

## 一、项目背景与研究目标

### 1.1 项目背景

交易猫平台计划对**王者荣耀**和**鸣潮**两款热门游戏的商品详情页进行UI界面改版。针对每款游戏，设计团队已产出3套候选设计方案。为确保改版方向符合用户预期、提升用户体验与转化效率，需从用户视角系统评估各方案的偏好表现及背后原因，为最终方案选择与后续优化提供数据支撑。

### 1.2 研究目标

本研究旨在达成以下核心目标：

1. **量化评估方案偏好**：通过定量问卷测量用户对3套候选方案的整体偏好度及关键维度评分，识别最优方案
2. **深挖偏好驱动因素**：通过定性访谈挖掘用户喜欢/不喜欢各方案的具体原因，聚焦页面信息呈现、布局层级、视觉风格等核心要素
3. **明确优化方向**：基于用户反馈提炼可落地的设计优化建议，指导后续迭代
4. **验证设计假设**：检验设计团队对各方案优劣势的预判是否与用户真实感受一致

## 二、研究设计与方法论

### 2.1 研究思路

采用**"定量验证 + 定性深挖"**的混合研究方法，分两阶段推进：

- **第一阶段（定量）**：通过线上问卷快速收集大样本数据，验证各方案的相对偏好度及统计显著性
- **第二阶段（定性）**：基于定量结果筛选典型用户进行1v1深访，深入探究偏好背后的心理机制与具体痛点

该设计兼顾广度与深度，既保证结论的代表性，又确保洞察的可操作性。

### 2.2 定量研究设计

#### 研究对象

近1个月内浏览过**王者荣耀**或**鸣潮**商品详情页的交易猫平台用户。

#### 样本量规划

| 游戏 | 样本量 | 说明 |
|------|--------|------|
| 王者荣耀 | N ≥ 500人 | 独立样本，单独投放问卷 |
| 鸣潮 | N ≥ 500人 | 独立样本，单独投放问卷 |
| **合计** | **N ≥ 1000人** | 两款游戏问卷分别投放，不交叉 |

#### 问卷核心内容

**1. UI方案偏好评估**
- 展示3套候选方案截图/原型，请用户排序或打分
- 各方案在关键维度的评分（如：信息清晰度、视觉吸引力、操作便捷性、整体美观度等）
- 开放题：最喜欢/最不喜欢的方案及原因简述

**2. 用户基本信息**
- 人口统计学变量：年龄、性别、职业、学历、所在城市等级
- 游戏行为特征：游戏时长、消费频次、近期购买记录
- 联系方式：用于后续定性招募（需获得用户授权）

#### 质量控制

- 设置注意力检测题（如："请选择'非常同意'"）剔除无效答卷
- 限制答题时长下限（如少于60秒视为无效）
- IP去重，同一设备仅允许提交一次

### 2.3 定性研究设计

#### 研究对象

从定量问卷受访者中筛选符合以下条件的用户：

- 近1个月内浏览过王者荣耀或鸣潮商品详情页
- 年龄 ≥ 18岁
- 愿意参与线上1v1视频访谈（时长约30-45分钟）
- 在定量阶段对方案有明显偏好或强烈意见（极端用户优先）

#### 样本配额

| 游戏 | 样本量 | 配额要求 |
|------|--------|----------|
| 王者荣耀 | 6人 | 尽可能覆盖不同职业、年龄段、性别 |
| 鸣潮 | 6人 | 尽可能覆盖不同职业、年龄段、性别 |
| **合计** | **12人** | 两款游戏独立招募，不交叉 |

> **注**：以上配额为预估值，实际完成情况将根据用户报名意愿及招募进展动态调整。若某类游戏招募困难，可适当放宽职业分布要求，优先保证样本数量。

#### 访谈重点

1. **方案偏好归因**
   - 对各方案的直观感受（第一眼印象）
   - 喜欢/不喜欢各方案的具体原因（聚焦：信息呈现逻辑、布局层级合理性、视觉风格匹配度、交互流畅性等）
   - 在实际使用场景中的潜在问题（如：查找商品信息、对比价格、判断可信度等）

2. **优化建议收集**
   - 理想中的商品详情页应具备哪些要素
   - 对现有方案的具体改进建议（如：增加/删减某些信息模块、调整颜色/字体、优化按钮位置等）
   - 与其他电商平台/游戏交易平台的对比体验

3. **深层需求挖掘**
   - 购买游戏商品时的核心决策因素
   - 对平台信任感的来源与顾虑点
   - 影响转化转化的关键障碍

#### 访谈执行方式

- **形式**：线上1v1视频访谈（腾讯会议/钉钉会议等）
- **时长**：每人30-45分钟
- **记录**：全程录音+屏幕共享录制，访谈后24小时内完成转录与编码
- **激励**：提供50-100元等价礼品卡或平台优惠券作为酬谢

## 三、项目时间规划

### 3.1 总体周期

项目预计需要 **31~36个工作日** 完成，具体进度将根据用户招募情况及数据回收效率动态调整。

### 3.2 详细阶段分解

| 阶段 | 主要工作内容 | 预计工作日 | 关键交付物 |
|------|--------------|------------|------------|
| **前期准备** | 研究方案确认、利益相关方对齐 | 1天 | 终版研究方案、项目启动会纪要 |
| **定量阶段** | | | |
| ├─ 问卷设计与编程测试 | 问卷题目撰写、逻辑跳转设置、小范围预测试 | 1天 | 问卷终稿、测试报告 |
| ├─ 问卷投放与数据回收 | 多渠道投放、实时监控回收进度、必要时追加投放 | 5天 | 原始数据集（≥1000份有效问卷） |
| ├─ 数据清洗与分析 | 无效样本剔除、描述性统计、交叉分析、显著性检验 | 2天 | 定量分析报告、关键发现摘要 |
| **定性阶段** | | | |
| ├─ 访谈大纲设计 | 基于定量结果细化访谈提纲、内部评审确认 | 1天 | 访谈大纲终稿 |
| ├─ 用户招募 | 从定量受访者中筛选、电话/短信邀约、确认参与意向 | 4天 | 确认参与的用户名单（12人） |
| ├─ 深访执行与个案整理 | 开展1v1访谈、录音转录、初步编码与主题提取 | 10~15天 | 访谈逐字稿、初步洞察笔记 |
| **分析与汇报** | | | |
| ├─ 整合分析与报告撰写 | 定量+定性数据三角验证、提炼核心洞察、形成设计建议 | 5天 | 完整研究报告（含数据图表、用户原声、设计建议） |
| ├─ 结果讨论与调整 | 与设计团队/业务方汇报、收集反馈、修订报告 | 2天 | 终版研究报告、汇报PPT |

### 3.3 风险与应对

| 风险点 | 影响程度 | 应对措施 |
|--------|----------|----------|
| 定量问卷回收不足 | 高 | 预留2天缓冲期；扩大投放渠道；增加激励力度 |
| 定性用户招募困难 | 中 | 提前建立备选用户池；放宽部分配额要求；延长招募周期 |
| 用户访谈爽约 | 中 | 每类游戏超额招募2-3人作为备份；访谈前24小时再次确认 |
| 数据质量不佳 | 低 | 严格设置质控题目；人工复核开放题答案；必要时补充少量样本 |

## 四、预期成果与价值

### 4.1 交付物清单

1. **定量研究报告**：包含样本特征、方案偏好排名、各维度评分对比、显著性检验结果
2. **定性洞察报告**：包含用户原声摘录、典型痛点归纳、优化建议汇总
3. **综合研究报告**：整合定量+定性发现，形成最终结论与设计指引
4. **汇报材料**：精简版PPT，用于向管理层/设计团队同步关键发现

### 4.2 业务价值

- **降低试错成本**：通过前置用户验证，避免上线后大规模返工
- **提升转化效率**：基于用户真实偏好优化页面设计，预期提升点击率与转化率
- **沉淀方法论**：形成标准化的UI改版研究流程，可复用于后续其他游戏品类

## 五、附录

### 5.1 术语说明

- **UI方案**：指商品详情页的视觉界面设计方案，包括布局、色彩、字体、图标等元素
- **深访**：深度访谈（In-depth Interview），一种半结构化的定性研究方法
- **显著性检验**：统计学方法，用于判断不同方案间的偏好差异是否具有统计意义
